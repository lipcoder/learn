# 6.4

## Q:

我们经常选择一个方法，并且在同一个表达式里执行，比如常见的p.Distance()形式，实际上将其分成两步来执行也是可能的。p.Distance叫作“选择器”，选择器会返回一个方法“值”->一个将方法（Point.Distance）绑定到特定接收器变量的函数。这个函数可以不通过指定其接收器即可被调用；即调用时不需要指定接收器
讲解一下go圣经里面这句话,我能理解在go里面方法是可以与自己定义的类型来组合的,但是上面这段话我看的一知半解

## A:

### 1. 先定义一个类型和方法

```go
type Point struct {
    X, Y float64
}

func (p Point) Distance(q Point) float64 {
    // 计算两点间距离
    return math.Hypot(p.X-q.X, p.Y-q.Y)
}
```

下面是通常的做法

```go
p := Point{1, 2}
q := Point{4, 6}
d := p.Distance(q)   // 一步完成：指定接收器 p，参数 q
```



------

### 2. 什么是“分成两步执行”？

Go 允许你先 **只选择方法，而不立即调用它**：

go

```go
// 第一步：通过选择器 p.Distance 得到一个“方法值”
methodValue := p.Distance   // 注意：没有括号，不是调用

// 第二步：调用这个“方法值”，传入剩余的参数
d := methodValue(q)         // 不再需要写 p，因为 p 已经绑定了
```



这里的 `p.Distance` 就是 **选择器（selector）**，它返回的是一个 **方法值（method value）**。

------

### 3. 方法值的本质

`methodValue` 的类型是 `func(Point) float64`（一个普通函数）。
但它和普通的 `Point.Distance` 方法不同：

- **普通方法** `Point.Distance` 需要两个输入：接收器 `p` 和参数 `q`（即 `func(Point, Point) float64`）。
- **方法值** `p.Distance` 已经 **把接收器 `p` 固定在了函数内部**，所以它只需要参数 `q`（即 `func(Point) float64`）。

也就是说，当你写 `p.Distance` 时，Go 会生成一个 **闭包**，它记住当前的接收器 `p`。之后你调用它，就不需要再提供接收器了。

## ps1: 闭包:

**闭包 = 函数 + 它引用的外部变量**

普通函数只能使用自己的参数和全局变量。
闭包则像是一个“会记忆”的函数：它内部使用了**不属于它自己的、但来自外部作用域的变量**，并且把这些变量的当前值“捕获”到函数内部，随身携带。

### 一个经典的闭包例子（不涉及方法）

```go
func Adder(x int) func(int) int {
    // 返回的匿名函数引用了外部的 x
    return func(y int) int {
        return x + y
    }
}

add5 := Adder(5)   // add5 是一个闭包，它记住了 x = 5
fmt.Println(add5(3)) // 输出 8，因为内部是 5 + 3
```



这里的 `add5` 就是闭包：它里面有一个“藏起来的” `x`，值是 5。
当你调用 `add5(3)` 时，只需要给 `y`，`x` 已经自己记住了。

## ps2:“基准点”

你提到的“基准点”正是这个被捕获的 `p`。

`Distance` 方法原本要计算“从接收器点到另一个点的距离”。
如果你写 `p.Distance` 得到一个方法值，那就相当于你固定了“起点”是 `p`，这个起点就是**基准点**。

之后你拿着这个方法值，只需要告诉它“终点是什么”，它就能算出相对于那个固定基准点的距离。

**换句话说：**

- `Point.Distance(p, q)` → 两个点都显式给出
- `p.Distance(q)` → 显式给出基准点 `p` 和终点 `q`
- `methodValue := p.Distance` → 基准点 `p` 被锁在闭包里；然后 `methodValue(q)` 只需给终点

------

## ps3. 方法值已经将方法的“类型变量”记录下来了？ 应该叫接受器变量

你这里说的“类型变量”可能是指 **接收器变量**（即具体的 `p` 这个值）。
**答案：是的，完全正确。**

方法值通过闭包机制，把接收器变量的**当前值**（或引用）保存下来了。
但注意：它记录的是**具体的值/引用**，而不是类型。类型信息在编译时已经确定（`p` 是 `Point` 类型），闭包捕获的是运行时的实例。

可以验证一下：

```go
p := Point{1, 2}
methodValue := p.Distance

p = Point{10, 20}   // 修改 p
fmt.Println(methodValue(q))   // 结果仍然基于最初的 {1,2}，不是 {10,20}
```

因为闭包在创建时就已经把 `p` 的值（`{1,2}`）拷贝保存了。
如果你希望闭包能反映后续对 `p` 的修改，需要让接收器是指针类型：

```go
type Point struct { X, Y float64 }
func (p *Point) Distance(q Point) float64 { ... }

p := &Point{1, 2}
methodValue := p.Distance   // 捕获的是指针 p 的值（即地址）
p.X = 10                    // 修改原对象
fmt.Println(methodValue(q)) // 现在会看到基于 {10,2} 的结果
```
